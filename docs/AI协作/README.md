# AI 协作

本目录存放 AI 协作过程材料。

这里记录任务、咨询、反馈和阶段报告，不替代长期规范、正式方案或代码测试。

## 目录结构

```text
docs/AI协作/
├── README.md
├── 协作规范.md
├── 本地Agent/
│   ├── 进行中/
│   └── 归档/
├── 外部咨询/
│   ├── 进行中/
│   └── 归档/
└── 用户反馈/
    ├── 缺陷/
    ├── 改进/
    └── 计划/
```

## 分类说明

| 分类 | 用途 |
| ---- | ---- |
| `本地Agent/` | Codex、OpenCode、子 Agent 等本地可执行任务与报告 |
| `外部咨询/` | 需要复制粘贴给外部 AI、专家或搜索工具的问答材料 |
| `用户反馈/` | 人类实际使用、测试和验收中提出的问题与建议 |

## 生命周期

| 状态 | 放置位置 |
| ---- | ---- |
| 正在处理 | `本地Agent/进行中/` 或 `外部咨询/进行中/` |
| 已完成但需追溯 | `本地Agent/归档/` 或 `外部咨询/归档/` |
| 用户反馈 | 按类型长期放在 `用户反馈/缺陷/`、`用户反馈/改进/`、`用户反馈/计划/` |

## 与其他目录的关系

| 内容 | 应放位置 |
| ---- | ---- |
| 临时任务、咨询、阶段报告 | `docs/AI协作/` |
| 当前长期规范 | `docs/规范/` |
| 翻译准确性审核 | `docs/review/` |
| 复杂缺陷根因与回归 | `docs/bugfix/` |
| 方案设计 | `docs/superpowers/specs/` |
| 实施计划 | `docs/superpowers/plans/` |

协作材料形成长期规则后，必须沉淀到 `docs/规范/` 或 `AGENTS.md`。
