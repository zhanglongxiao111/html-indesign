# AI 协作规范

本规范只定义协作文档怎么放、怎么命名、怎么流转。

## 1. 适用范围

适用于 `docs/AI协作/` 下的三类材料：

- 本地 Agent 任务和报告。
- 外部咨询问答。
- 用户反馈记录。

## 2. 本地 Agent

用于记录可自动化执行的任务，例如子 Agent 审核、OpenCode 分析、Codex 辅助检查。

路径：

| 状态 | 路径 |
| ---- | ---- |
| 进行中 | `docs/AI协作/本地Agent/进行中/YYYY-MM-DD_主题/` |
| 已归档 | `docs/AI协作/本地Agent/归档/YYYY-MM-DD_主题/` |

推荐文件名：

| 文件 | 用途 |
| ---- | ---- |
| `任务_主题_Agent名.md` | 发给 Agent 的任务说明 |
| `报告_主题_Agent名.md` | Agent 返回的结论、证据和建议 |

## 3. 外部咨询

用于记录需要人工复制粘贴到外部 AI、专家或搜索工具的问答材料。

路径：

| 状态 | 路径 |
| ---- | ---- |
| 进行中 | `docs/AI协作/外部咨询/进行中/YYYY-MM-DD_主题/` |
| 已归档 | `docs/AI协作/外部咨询/归档/YYYY-MM-DD_主题/` |

推荐文件名：

| 文件 | 用途 |
| ---- | ---- |
| `提问_主题.md` | 给外部工具的问题 |
| `回复_主题.md` | 外部工具的回复 |
| `追问_主题.md` | 后续追问 |
| `回复_追问_主题.md` | 追问后的回复 |

## 4. 用户反馈

用于记录人类用户实际使用、测试和验收中的反馈。

路径：

| 类型 | 路径 |
| ---- | ---- |
| 缺陷 | `docs/AI协作/用户反馈/缺陷/` |
| 改进 | `docs/AI协作/用户反馈/改进/` |
| 计划 | `docs/AI协作/用户反馈/计划/` |

文件名统一使用：

```text
YYYY-MM-DD_简短描述.md
```

## 5. 写作要求

每篇协作文档开头至少写清楚：

- 背景。
- 目标。
- 当前结论。
- 相关文件路径。
- 是否需要沉淀为长期规范。

## 6. 禁止事项

- 不要把协作文档放到 `docs/` 根目录。
- 不要把临时报告当作长期规范。
- 不要把正式规范写进 Agent 报告后就结束。
- 不要记录客户文档内容、客户名称或私有资产完整路径。
- 未确认完成前，不要删除协作文档。
