# 汇总：架构硬化核心代码膨胀审计

## 背景

用户关注本轮架构硬化重构后核心代码净增是否可能来自老代码未退役。本文由主控汇总 7 个只读审计方向，并在 Follow-up Tasks 1-7 修复完成后更新为 closeout 状态。

## 当前结论

截至 Task 7 closeout，原始审计中的 P1/P2 半收口问题已经完成闭环；当前边界如下：

1. **核心代码增长不是旧 `paged-html` / `indesign-reverse` / public facade 没退役。**
   - `src/paged-html`、`src/indesign-reverse` 不存在。
   - public API 不暴露 `pagedHtml`、`indesignReverse`、`historicalTemplate`。
   - G5 retired naming、G8 orphan modules、public API 退役测试继续作为护栏。

2. **协议退役字段由 lifecycle 统一治理。**
   - `items[].type`、`items[].effects`、`items[].textFrameStyle` 已进入 retired model path lifecycle。
   - `migration.from` 只记录迁移来源，不能替代 retired lifecycle。
   - 生成文档 `docs/规范/PROTOCOL_FIELD_REGISTRY.md` 只呈现 registry 事实，不作为第二字段事实源。

3. **反向作者包审核门禁已统一到 `src`。**
   - reverse export、plugin reverse export 和 E2E reverse roundtrip 使用同一组 `src/writers/html/audit/` 作者包审核语义。
   - 反向视觉证据补全已从脚本迁入 `src/writers/html/audit/reverse-visual-evidence.js`。
   - 脚本和插件只负责参数、路径、输出登记和退出码。

4. **架构 guardrail 不允许静默扩 baseline。**
   - G1/G2/G4/G5/G6/G7/G8 baseline 都通过 ratchet 约束。
   - G4 fail-closed 检查验证真实失败行为，不再只检查测试名。
   - guardrail 扫描排除本地 scratch 和测试输出，避免并发临时目录导致假失败或假绿。

5. **共享 helper 语义已收敛为单一可执行来源。**
   - 资产路径、table width、border visibility、instruction text normalization 等语义族由共享模块或明确归属模块提供 canonical 实现。
   - G6 按语义族检查重复实现，防止通过改名保留同义 helper。

6. **`.superpowers/` 是本地 scratch only。**
   - `.gitignore` 忽略 `.superpowers/`。
   - 已跟踪的 `.superpowers/sdd/*.md` 过程材料已从版本库移除。
   - 需要长期保留的审计材料位于 `docs/AI协作/本地Agent/进行中/2026-07-08_架构硬化核心代码膨胀审计/`。
