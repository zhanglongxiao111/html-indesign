# 任务：测试增长质量审计

## 背景

本轮重构测试净增约 8000 行。需要判断测试增长是有效 guardrail 和回归覆盖，还是空测试、重复断言、快照噪音或为了 baseline 过关的表面测试。

## 目标

审计新增测试质量，特别是 `test/architecture/`、`test/protocol/`、`test/indesign-to-html/`、`test/html-to-indesign/` 的新增覆盖是否真实验证行为。

## 范围

- `test/architecture/`
- `test/protocol/`
- `test/indesign-to-html/`
- `test/html-to-indesign/`
- `test/shared/`
- 与测试直接关联的 source 文件可只读查看

## 硬约束

- 只读审计，不得修改代码、测试、配置、git index 或 HEAD。
- 只允许写自己的报告：`docs/AI协作/本地Agent/进行中/2026-07-08_架构硬化核心代码膨胀审计/报告_测试增长质量审计.md`
- 不得调用子 agent。
- 不跑真实 InDesign E2E。
- 严格执行项目原则：测试必须验证真实行为；guardrail 不能靠空 baseline 或弱扫描假绿；兜底和假成功必须 fail closed。

## 建议命令

- `git diff --numstat 5d4757a6f7efbe8c8410f816d462a90813567fb7..HEAD -- test`
- `rg "test\\(|assert\\.|assert\\.equal|assert\\.throws|baseline|exemptions|TODO|skip|only" test/architecture test/protocol test/indesign-to-html test/html-to-indesign test/shared`
- `node --test test/architecture/`
- 可抽查 targeted tests，但要记录命令和结果。

## 报告要求

报告必须包含：

- 当前结论：测试增长有效 / 存在低价值测试 / 存在 guardrail 假绿风险。
- 证据：文件行号、命令结果摘要。
- 风险分级：P0/P1/P2/none。
- 可删候选：重复或低价值测试列表。
- 不可删理由：对主要新增 guardrail 测试说明保留价值。
- 是否需要沉淀为长期规范。
