# 兜底字段排查任务

## 背景

仓库：`D:\AI\html-indesign`

当前项目原则是固定语义 HTML 与 InDesign 的双向翻译。语义字段必须是白名单协议，不能为了兼容历史字段而在运行时做模糊读取。刚刚已把 PDF 置入页码收紧为唯一字段 `data-id-pdf-page`：旧 `data-id-page` 只能作为无效观察字段或迁移问题，不得参与编译，也不得静默兜底为第一页。

## 任务目标

请排查仓库中还存在哪些类似风险：

- 同一个语义事实存在多个字段名，并用 `a || b || c`、`fallback`、`legacy` 等方式模糊读取。
- 旧字段、别名字段、历史字段仍参与正向编译或反向结构化输出。
- 缺少关键协议字段时，代码静默填默认值继续执行。
- 文档中仍表达“兼容读取旧字段”或“默认兜底”的长期规则。

## 边界

- 只做排查和清单，不修改代码。
- 不要把所有普通默认值都列为问题；只列会造成语义事实丢失、字段混淆、反向回环漂移或假成功的兜底。
- 区分“必须清理”“需要确认”“可保留的兼容边界”。
- 每个问题给出文件路径、关键代码或文档片段、风险说明、建议清理方式。
