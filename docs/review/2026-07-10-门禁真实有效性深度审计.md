# 门禁真实有效性深度审计

- 日期：2026-07-10
- 触发：用户质疑"无丢失"门禁全绿但真实丢失存在（页码母版对象丢失、几何对象 143→109 未拦截），要求对门禁体系做深度审核。
- 方法：四路并行代码审计（内容库存对比器、effective-diff 分级器与总门禁、视觉事实采集层、E2E 门禁接线）+ 三个实证案例复现（基于 `test/workspace/indesign-e2e-20260710-143013/` 真实运行产物）。
- 结论级别：**门禁体系当前不能担保"无丢失"**。"报告 ok:true 但实际有丢失"不是偶发，而是三条链路叠加的默认结构性结果。

## 1. 实证案例（全部来自同一次全绿 E2E 运行）

### 案例一：母版页码整体丢失，内容库存门禁全绿

- 源码 `test/fixtures/e2e/architecture-report/pages/00-cover.html:19` 声明页码 `<span id="cover-folio" data-id-placement="parent-page-furniture">00</span>`。
- 正向构建正确：InDesign 母版"汇报母版"上存在 `report-folio` 对象，反向快照也完整捕获（`reverse-snapshot.json` 中含文本 "00" 与完整标签）。
- 反向作者包 `reverse-html/author/` 中该对象**完全消失**：无 parent-pages 文件，页面仅存 `data-id-parent-page` 引用属性。HTML 写出器不写回母版内容。
- `content-inventory-report.json` 依然 `ok:true`、texts 81/81——因为母版家具被四个盘点维度**显式排除**（见 C-1）。
- 后果：拿该作者包再次正向构建，第二代 InDesign 将真实丢失页码。

### 案例二：渐变填充静默丢弃，全链路零警告

- fixture 唯一渐变 `styles/pages.css` `.veil`（封面渐变纱罩，linear-gradient 三段 rgba 色标）。
- 编译产物 `instructions.json` 中 `cover-hero-veil` 是**无任何填充字段的矩形**；其声明的 `data-id-object-style="cover-veil"` 对应样式在样式表中**根本没有被合成出来**。
- 浏览器：标题区有白色渐变纱罩保证可读性；InDesign：无色矩形。视觉偏差真实存在，任何报告无记录。
- 对照：rgba 纯色填充处理正确（`指标卡片` 样式带 `fillColor` + `fillOpacity: 0.92` + `strokeAlignment: "inside"` + `cornerRadius: "4pt"`），说明纯色链路成熟、渐变链路缺失。

### 案例三：两个检测器都命中真实丢失，E2E 依然 exit 0

- 同一次运行：`structure-signature-report.json` `ok:false`（32 条 `STRUCTURE_NODE_MISSING`，metrics 表 thead/tbody/tr/th/td 作者节点全丢）；`source-roundtrip-report.json` `ok:false`（7 页 `TAG_SEQUENCE_CHANGED` + 7 页 `TEXT_CHANGED`，即页码丢失）。
- `e2e-result.json` 顶层 `ok:true`，进程 exit 0。两个命中都被默认降级为 advisory。

## 2. 失效分类总览

门禁失效分四类，按"从源头到出口"排列：

```text
一类 采集层缺失   —— 快照里没有这个事实，任何比对天生失明
二类 对比器盲区   —— 事实在，但对比器不比、比错或比对范围排除
三类 接线失效     —— 比出来了 ok:false，但不参与最终判定/退出码
四类 考卷盲区     —— 唯一真实 fixture 从未考过这些属性，门禁行为未知
```

## 3. 一类：采集层缺失

依据"视觉属性 × 五阶段"覆盖矩阵审计（正向采集 → 语义模型 → 正向写出 → 反向采集 → 反向写出）。

| 编号 | 属性 | 缺口 | 严重度 |
| ---- | ---- | ---- | ---- |
| S-1 | 渐变填充 | 语义模型无 gradient 字段；radial 不解析、多色标 `GRADIENT_COLOR_UNSUPPORTED` 丢弃、单色塌成纯色 swatch；反向 JSX `reverseColor` 对渐变色板返回 null，InDesign 彩色渐变完全不采集（`hi_reverse_styles.jsxinc:287-302`） | 高 |
| S-2 | 阴影/发光/斜面等效果 | 反向 `reverseEffects` 只导出 gradientFeather，投影/内阴影/发光/斜面全部不读（`hi_reverse_effects.jsxinc:3-7`）；正向 box-shadow 只进 `unsupported` 警告、无字段承接 | 高 |
| S-3 | 旋转/切变 | **单向能力**：正向能解析 `rotate()` 写入 `rotationAngle` 并执行（`instruction-writer.js:682`、`hi_items.jsxinc:124`），反向 `reversePageItem` 不读回旋转角（`hi_reverse.jsxinc:199-226`），往返必丢且不可比；`rotationAngle` 未登记进 protocol 注册表；scale/skew 全程无支持 | 高 |
| S-4 | JSX 读失败静默 null | `hi_reverse*.jsxinc` 大量 `catch(_){} return null`：属性读取抛错与"确实没有"不可区分，双侧 null 相等假通过（"双侧空值抵消"的根因之一） | 高（系统性） |
| S-5 | 逐角圆角/圆角类型 | 反向只采 `topLeftCornerRadius` 单角，斜切/花式/内凹类型不导出 | 中 |
| S-6 | 表格结构签名 | 反向导出了 rowCount/columnCount/列宽/行高/表样式，但 `itemSignature` 不纳入（`reverse-snapshot-structure.js:196-227`），表格结构变化不可见（文字内容经 text 仍受 P0 保护） | 中 |
| S-7 | 文本视觉样式 | `textStyle/textRuns/firstLineFont` 导出但不比对：字体/字号/颜色/字重变化 effective-diff 不可见 | 中 |
| S-8 | SVG 路径命令 | 正向 `parsePathPoints` 只认 M/L/C/Z，Q/S/T/A（二次贝塞尔/平滑/圆弧）不解析，锚点漏采 | 中 |
| S-9 | CSS 线形 | CSS 边框只采 border-style，无 dash-array/linecap/linejoin/箭头（SVG 线全链路保真，CSS 画的线丢精细线形）；反向 dash 数组不捕获、仅存 strokeType 名 | 中 |
| S-10 | 边框 rgba alpha | `compileBorderEdge` 只取颜色 hex，边框半透明丢 alpha | 低 |
| S-11 | 描边对齐可视层 | 模型/指令/反向均保真，但视觉 HTML 与 SVG 均不渲染（CSS 无此能力），仅作者包 `data-id-stroke-alignment` 保真——视觉几何门禁对 inside/outside 差异无感 | 低 |

**判定良好的属性**：透明度（元素 opacity、fill 独立 alpha、SVG opacity 全链路贯通）、三次贝塞尔矢量曲线（锚点+控制柄+pointType 往返保真）、图片裁切与适配（fit/crop/pdfCrop/contentOffset 等全比）、混合模式与线端箭头（机制在，条件导出）、单角圆角、描边对齐（数据层）。

## 7. 加固方案

原则：先让门禁**不再说谎**（三类），再让门禁**能看见**（一类/二类），最后**补考卷**（四类）。硬化门禁会让当前 E2E 变红（页码丢失、表格节点丢失是真实缺陷）——按"失败要早于假成功"，这是期望行为；同批修复根因（母版写回、表格 id 保留），不得用扩预算掩盖。

### 第一批：接线与判定收口（让绿灯可信）

1. `result.ok` 改为对关键报告 ok 的显式聚合，废除硬编码 true（W-1）。
2. structure-signature 升为默认硬门槛，丢失类错误（`STRUCTURE_NODE_MISSING`）无条件计入 errors（W-2/C-9）。
3. source-roundtrip 的纯丢失类错误（tag 序列缩短、text 变化、resource 变化）进入聚合 ok；`strictSourceRoundtrip` 死参数按退役纪律当场清除或接通（W-2/W-3）。
4. effective-diff：带非空 text/placedAsset 的 `ITEM_MISSING/ITEM_EXTRA` 升 P0（C-11）；预算缺失默认 0 或显式报错、坏 baseline 直接 fail（C-12）；稳定性审计校验结构（C-13）。
5. conversion-gate：期望门在 case 显式声明，缺失报告显式 SKIPPED 并可配为硬失败（W-5）；reverse-visual 断言 `compared>0`（W-6）。
6. 反向导出 CLI 与插件与 E2E 共用同一严格度；插件 lint 失败映射 `status:'error'`；fail-open 形状改 `!== true`（W-4/W-10/W-11）。

### 第二批：对比器补盲（让硬门槛覆盖真实丢失面）

7. content-inventory 新增 furnitureDigest（按 `data-id-parent-page-source-id` 归一）纳入硬门槛（C-1）——**须与"母版内容写回作者包"的写出器修复同批落地**，否则门禁立即红。
8. geometryItems/texts/resources 计数守恒设硬门槛；itemRoles 按角色计数纳入比较或删除死字段（C-2/C-3）。
9. 资源身份改内容 hash（复用 source-roundtrip 已有 sha256）、多重集计数、取消 previews 整体豁免（C-4）。
10. 文本盘点改为遍历页面全部可见文本节点；无丢失层用码点级归一，CJK 折叠只留给 advisory 层（C-5/C-6）。
11. 逐页双空断言：expected 非空而 actual 空即丢失，双侧同空记 warning（C-7）。
12. structure-signature 纳入 classList 比较 + actual 多出节点统计（C-9）。

### 第三批：采集层补盲（配合语义保留升级主线）

13. 反向 JSX：导出旋转/切变角、投影等效果、渐变（type/angle/stops）、四角圆角+类型、dash 数组；表格结构纳入 signature（S-1..S-6）。
14. JSX 读失败与"确实没有"区分：失败记 `readError` 标记，审计端把"读失败"当审核项而非 null（S-4）。
15. 正向：渐变字段登记 protocol 注册表后接入语义模型与写出（能力不足前至少显式报 `GRADIENT_UNSUPPORTED` 审核项，禁止静默丢弃）；作者声明的 `data-id-object-style` 合成失败必须报错（案例二两个缺陷）。
16. `rotationAngle` 登记注册表，反向读回，闭合往返（S-3）。

### 第四批：考卷补齐

17. fixture 新增"视觉属性考卷页"：虚线、箭头、SVG 曲线（含 Q/A 命令）、渐变、阴影、旋转、逐角圆角、半透明边框——新考卷会把一批已知缺陷考红，作为第二、三批修复的验收基准。
18. parent-furniture 指标接入 conversion-gate 并设阈值（W-8）。

## 8. 与"同事可用"总计划的关系

本审计对应总计划的"第 0 步：地基与信任"。在门禁按本方案收口之前，语义重建算法（R1–R9）的任何"进步"都无法被可信度量——先修门禁，再谈算法。
