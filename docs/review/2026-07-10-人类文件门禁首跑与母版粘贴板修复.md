# 人类文件门禁首跑与母版粘贴板修复

- 日期：2026-07-10
- 场景：`npm run e2e:indesign -- --indd <document.indd> --run-dir <local-test-workspace>` 一键回环门禁，首次在真实人类 InDesign 文档上运行
- 样本：2026-06-03 计划文档（`docs/superpowers/plans/2026-06-03-human-indesign-lossless-roundtrip.md`）使用的同一份 47 页真实汇报文档，NAS 原位引用，回环输出写本地 `test/workspace/`
- 关联：`docs/review/2026-07-10-门禁真实有效性深度审计.md`（本次运行是该轮门禁加固后的第一次人类文件实测）

## 1. 首跑红灯：母版粘贴板暂存件被虚构成页面家具

### 1.1 现象

真实 InDesign 构建失败，16 个执行错误，全部为：

```text
PARENT_PAGE_ITEM_OVERRIDE_FAILED  Error: 此请求的对象无效。
```

集中在两个母版文本框（源 id `1033`、`1288`），分布在 16 个使用对应母版的页面；同母版的其他文本项覆盖全部成功。

### 1.2 根因

- 原始文档的两个母版上各有一个文本框被设计师停放在**母版粘贴板**上（y≈-132pt，整体在页面上边缘之外）。InDesign 中这种母版项不会显示在任何文档页上，`override(page)` 对它必然抛"此请求的对象无效"。
- 家具回写（`shouldWriteParentPageItem`）没有"母版项是否落在页面内"的判断；observation 模式下 InDesign 适配器给母版项生成 `source: 'indesign-reverse'` 的 accepted 标签，命中 `hasAcceptedItemProtocolLabel` 早退分支被无条件放行。
- 结果：粘贴板暂存件被写成 47 页中 16 页的页面家具 span（虚构了原文档从未在页面上显示的内容），正向编译又为每个实例发出 override 指令，执行时全部失败。

### 1.3 修复（同轮收口）

沿用 fixture 已验证的家具 span 回路承载，不为绕过浏览器另建样式合成路径：

| 层 | 文件 | 内容 |
| --- | --- | --- |
| 共享几何 | `src/shared/geometry.js` | 新增 `boundsIntersectPage(bounds, pageSize)`，页尺寸未知时保持原行为 |
| 反向写出器 | `src/writers/html/author-package-writer.js` | 与页面无交集的母版项不再写给每一页，只写进使用该母版的第一页，标记 `data-id-placement="parent-page-pasteboard"`；母版保真保留（重建文档母版仍有该框） |
| 属性写出 | `src/writers/html/author-node-attrs.js` | `addParentPageAttrs` 写出 placement 标记 |
| HTML 适配器 | `src/adapters/html/normalizer/snapshot-to-model.js` | `data-id-placement` 提为模型字段 `items[].placement` |
| 指令写出器 | `src/writers/indesign/instruction-writer.js` | pasteboard placement 的家具实例不发 override |
| 校验器 | `src/writers/indesign/instructions-validator.js` | 新增 `PARENT_PAGE_ITEM_OVERRIDE_OFF_PARENT_PAGE`：override 指向整体在母版页面外的母版项时编译期报错 |
| 协议注册表 | `src/protocol/fields/html-authoring.js` | `items[].placement` 补登 `pages[].items[]`、`pages[].parentPageItems[]`、`parentPages[].items[]` 模型路径；`PROTOCOL_FIELD_REGISTRY.md` 已重新生成 |
| 常量 | `src/semantic-model/parent-pages.js` | `PARENT_PAGE_PASTEBOARD_PLACEMENT` 与 `isParentPagePasteboardItem` 共享定义 |

新增测试 4 个（几何判断、pasteboard 一次性回写、override 跳过、校验器拦截），全量 `npm test` 1016/1016 通过。

### 1.4 复跑验证

- `PARENT_PAGE_ITEM_OVERRIDE_*` 错误 16 → 0，真实 InDesign 构建成功（47 页）。
- 作者包中 `parent-page-pasteboard` span 恰好 2 处，分别位于两个母版各自的第一个使用页，源 id 与母版项一致。

## 2. 复跑红灯：首轮三报告暴露的既有翻译链缺口

粘贴板问题清除后，门禁推进到首轮硬门槛（作者包 vs 重建文档回读包），三报告仍 fail。按根因族归类：

| # | 缺陷族 | 证据 | 规模 |
| --- | --- | --- | --- |
| 1 | 内建默认样式经回环被克隆改名：`[无]`/`[基本图形框架]`/`[基本段落]` 等在正向被建成去括号+哈希后缀的自定义样式，二轮回读不再识别为默认样式 | `STRUCTURE_NODE_CLASS_REMOVED/ADDED`（`ostyle--无-` vs `ostyle-无`、`ostyle--基本图形框架-` vs `ostyle-基本图形框架-21251102`） | 35+35 |
| 2 | synthesized 样式 token 两轮编号不稳定 | 同报告中 `synth-synth_object_028/029` 仅单侧出现 | 混入 #1 计数 |
| 3 | 家具回写顺序两轮不对称（第 1 页母版家具 order 漂移） | `STRUCTURE_NODE_ORDER_CHANGED` | 35 |
| 4 | 带 marker 的 SVG 矢量（箭头类标注）正向构建丢失 | `STRUCTURE_NODE_MISSING`（页 27×28 个、页 45×1 个）、`CONTENT_GEOMETRY_ITEMS_LOST`、geometryItems 690→661 | 29，属 P0 级对象丢失 |
| 5 | 文本强制换行/空格在回环中被合并（折叠文本差异） | `ROUNDTRIP_TEXT_CHANGED` | 2 页 |
| 6 | 置入资产 crop 几何与 PSD 预览派生文件命名随 id 漂移 | `ROUNDTRIP_INLINE_STYLE_CHANGED`、`ROUNDTRIP_RESOURCE_CHANGED` | 各 32 页 |

附带观察（warning 级，不拦截）：

- 复合字体名（如按用途命名的自定义复合字体）在执行机上不可用，`FONT_APPLY_FAILED`；属字体预检/映射范围，见 `docs/规范/FONT_POLICY.md`。
- 一个 PSD 的图层可见性应用失败（`PLACED_ASSET_LAYER_VISIBILITY_APPLY_FAILED: Error: length`）。

## 3. 结论

- `--indd` 一键门禁作为人类文件门槛**有效**：第一次实测就抓住一个当日新引入的家具回写缺陷（已收口），并把既有翻译链缺口从"感觉不对"变成六个有计数、有报告定位的缺陷族。
- 缺陷族 #4（矢量丢失）是内容级损失，优先级最高；#1/#2/#3 是两轮稳定性问题，决定 canonical 门槛能否闭合；#5/#6 需按可验证问题逐个修。
- 修复应按族立项走共享链路收口，禁止在单页或 fixture 端打补丁。
